解决Centos-Minimal版本安装之后，无法上网，无法与局域网内机器通信

发表于 2017-04-19 | 分类于 linux | | 阅读次数
解决Centos-Minimal版本安装之后，无法上网，无法与局域网内机器通信

Minimal Centos安装后配置网卡

因为minimal centos安装后默认它是不启动eth0网卡的，所以我们就要手动启动。

0.首先设置虚拟机桥接模式。

1.去配置文件/etc/sysconfig/network-script/ifcfg-eth0,把ONBOOT=no,改为yes。并配置固定的ip(IPADDR),网关GATEWAY：192.168.1.1 , 子网掩码NETMASK:255.255.255.0 ，BOOTPROTO=static 表示固定ip地址

2.重启network服务：service network restart

3.如要centos可以上网，更改/etc/resolv.conf配置DNS地址与本地DNS地址一致。(一般配置完网卡配置文件后，这里会自动更改)

4.ping 192.168.1.92  #测试是否可以与局域网内机器通信
5.ping baidu.com     #测试可不可以联网

如果上述方法不行，说明eth0网卡没有启动。可以使用dhclient eth0自动获取一个eth0的ip，并生成/etc/resolv.conf的servername。然后再ifconfig eth0设置启动eth0网卡。这时候获取到的ip可能不是我们固定的ip，则修改配置文件固定ip，重启network服务即可。
